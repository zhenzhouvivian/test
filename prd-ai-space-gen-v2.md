# 需求背景

当前「AI空间生成」入口存在两个问题：
1. **输入体验**：界面暗示文字和图片均为必填，实际上两者提供其一即可
2. **生成质量**：用户输入的短文字不足以驱动模型生成高质量场景；需接入 Dify 扩写流程，将短描述扩展为包含空间布局、材质、光线的高质量英文 prompt

## 用户故事

- 作为内容创作者，我希望只上传一张参考图就能生成3D空间，不被强制要求写文字
- 作为专业用户，我希望输入几句话后，系统帮我扩写成详细 prompt，让生成结果更接近预期
- 作为普通用户，我希望内容违规时收到明确提示，而不是任务失败后才知道原因

## 需求的目标

| 目标 | 指标 | 目标值 |
|------|------|--------|
| 降低输入门槛 | 仅上传图片（无文字）成功提交率 | 100% |
| 提升生成质量 | 使用 AI 扩写后的满意度提升 | A/B 测试后定基线 |
| 减少无效任务 | 提交时内容违规拦截率 | 100%（关键词检测）|

**不做**：不改变生成模型；不修改物体/空间重建入口。

## 需求详述

### 输入区重新设计 [P0]

**功能描述**：将现有分离的图片/文字输入区合并为一个统一输入框，样式参考聊天输入框。

**布局规格**：

| 区域 | 规格 |
|------|------|
| 图片区（左）| 52×52px 缩略图位，虚线边框，「+」图标，上传后显示预览图，悬停显示删除按钮 |
| 文字区（右）| flex:1，多行自增高，placeholder「输入文字或上传参考图，仅支持室内空间生成」|
| 底部工具栏 | 左：`✦ 自动填写` \| `☆ AI 扩写`；右：字数统计（有文字时显示）+ 发送箭头按钮 |

**发送按钮状态**：

| 条件 | 状态 |
|------|------|
| 图片和文字均为空 | disabled |
| 有图片 或 有文字（≥1字）| enabled |
| 合法检测失败后 | enabled（允许用户修改后重新提交）|

**AI 扩写按钮**：文字 < 5 字时 disabled；≥ 5 字时 enabled。

**验收标准**：
- [ ] 仅上传图片可提交，跳过文字检测
- [ ] 仅输入文字可提交
- [ ] 两者均空时发送按钮 disabled
- [ ] 合法检测失败后仅显示错误提示，不阻止再次点击发送

---

### AI 扩写（Dify 接入）[P0]

**功能描述**：底部工具栏新增「☆ AI 扩写」按钮，点击后先做合法检测，通过后调用 Dify 工作流，流式输出扩写结果。

**交互规则**：

| # | 规则 |
|---|------|
| 1 | 点击 AI 扩写 → 先触发合法检测遮罩，违规则提示并终止 |
| 2 | 检测通过 → 调用 Dify，文字框清空并流式输出扩写结果（打字机效果）|
| 3 | 扩写期间按钮变为 loading 状态，文字框只读 |
| 4 | 扩写完成 → 框内底部显示「✦ 已由 AI 扩写优化 · 撤销」；按钮变为「重新扩写」|
| 5 | 点「撤销」→ 恢复原始输入，清除扩写标注 |
| 6 | 扩写完成后用户手动编辑文字 → 扩写标注自动消失 |

**Dify 接口**：

| 参数 | 说明 |
|------|------|
| 输入 | `user_input`（用户原始文字）、`scene_type: "indoor"` |
| 输出 | 流式 SSE，英文 prompt，不超过 1000 字符 |
| 超时 | > 10s 无响应时停止流式输出，提示「扩写超时，请重试」|

**验收标准**：
- [ ] 扩写前必须先经过合法检测
- [ ] 流式输出效果正常（打字机）
- [ ] 撤销后恢复原始输入
- [ ] 扩写后手动编辑，标注消失

---

### 合法性检测 [P0]

**触发时机**：点击发送（有文字时）或点击 AI 扩写。
**当前方案**：前端关键词匹配（MVP），后续可升级为后端接口。
**不合规关键词**（初始版）：outdoor、室外、sky、天空盒、户外、exterior。
**仅图片无文字**：跳过检测，直接提交。

---

### 自动填写 [P1]

保留底部「✦ 自动填写」按钮，点击后向文字框填入示例描述，AI 扩写按钮同步激活。

## 非功能性需求

| 指标 | 要求 |
|------|------|
| 检测遮罩响应 | 点发送/扩写后 < 200ms 出现遮罩 |
| Dify 扩写超时 | 10s 无响应时降级提示 |
| 扩写输出上限 | 1000 字符（前端截断）|
| 兼容性 | 与现有创建页保持一致 |

---

## 数据及埋点需求

| 事件名 | 触发时机 | 关键属性 |
|--------|---------|---------|
| `ai_expand_clicked` | 点击 AI 扩写 | `input_length`, `has_image` |
| `ai_expand_succeeded` | 扩写完成 | `output_length`, `duration_ms` |
| `ai_expand_failed` | 扩写超时/报错 | `error_type` |
| `ai_expand_reverted` | 点击撤销 | — |
| `content_check_failed` | 检测未通过 | `trigger: "create"\|"expand"` |
| `generation_queued` | 任务提交成功 | `has_text`, `has_image`, `is_expanded` |
