# 需求背景

当前「AI空间生成」入口存在两个问题：
1. **输入体验**：界面暗示文字和图片均为必填，实际上两者提供其一即可，用户存在困惑
2. **生成质量**：用户输入的简短描述（几个词）往往不足以驱动模型生成高质量场景；需接入 Dify 扩写流程，将用户短描述自动扩展为包含空间布局、材质、光线的高质量英文 prompt

## 用户故事

- 作为**内容创作者**，我希望只提供一张参考图就能生成3D空间，不被强制要求写文字
- 作为**专业创作者**，我希望输入几句话后系统帮我扩写成详细的 prompt，让生成结果更接近预期
- 作为**普通用户**，我希望内容违规时收到明确提示，而不是等到任务失败才知道

## 需求的目标

| 目标 | 指标 | 目标值 |
|------|------|--------|
| 降低输入门槛 | 仅上传图片（无文字）成功提交率 | = 100% |
| 提升生成质量 | 使用 AI 扩写后的用户满意度 | 待 A/B 测试后定基线 |
| 减少无效任务 | 提交时内容违规拦截率 | 100%（基于关键词检测）|

**不做**：不改变生成模型本身；不修改物体/空间重建入口。

## 需求详述

### 需求1：AI 扩写（Dify 接入）[P0]

**功能描述**：文字输入区底部新增「✦ AI 扩写」按钮，点击后调用 Dify 工作流，将用户短文描述扩写为高质量英文 prompt，以流式方式输出到文字框。

**交互规则**：

| # | 规则 |
|---|------|
| 1 | 文字少于 5 个字符时，AI 扩写按钮 disabled |
| 2 | 点击 AI 扩写 → 先触发合法性检测（同创建时检测逻辑），违规则提示错误并终止 |
| 3 | 检测通过后，调用 Dify 接口，文字框内容清空并开始流式输出 |
| 4 | 流式输出期间按钮变为 loading 状态（「扩写中…」），文字框只读 |
| 5 | 扩写完成后，输入框底部显示「✦ 已由 AI 扩写优化 · 撤销」标注 |
| 6 | 点「撤销」恢复用户原始输入，清除扩写标注 |
| 7 | 扩写完成后按钮变为「重新扩写」，可再次触发 |
| 8 | 扩写后用户可继续手动编辑文字框，编辑后标注自动消失 |

**Dify 接口参数**：

| 参数 | 说明 |
|------|------|
| `user_input` | 用户输入的原始文字 |
| `scene_type` | 固定为 `"indoor"` |
| 输出格式 | 流式 SSE，纯文本英文 prompt |

**验收标准**：
- [ ] 文字 < 5 字时按钮 disabled
- [ ] 点击 AI 扩写先做合法检测，违规终止
- [ ] 检测通过后文字框流式展示扩写结果
- [ ] 扩写完成后显示撤销入口
- [ ] 撤销后恢复原始输入

---

### 需求2：输入区重新设计（图片/文字均可选）[P0]

**功能描述**：将现有输入区重新设计为「左：图片上传（可选）」+「右：文字输入（可选）」的左右布局，顶部增加说明文案，明确至少提供其一即可。

**布局规格**：

| 区域 | 规格 |
|------|------|
| 图片区（左）| 宽 180px，高与文字区对齐，虚线边框，角标「可选」|
| 文字区（右）| flex:1，多行输入框，角标「可选」，底部字数统计 + AI扩写按钮 |
| 顶部提示 | 「图片或文字描述，提供其中一个即可开始生成」|

**创建按钮状态**：

| 条件 | 创建按钮状态 |
|------|------------|
| 图片和文字均为空 | disabled |
| 有图片 或 有文字（≥1字）| enabled |
| 合法检测失败后 | enabled（允许用户修改后再次提交）|

**合法性检测（创建时触发）**：
1. 点击「创建」
2. 若无文字（仅图片），跳过检测，直接提交
3. 若有文字，显示卡片遮罩 + loading
4. 检测完成：通过则直接提交；违规则隐藏遮罩、显示红色错误提示，不提交任务

**室内限制提示**：顶部蓝色 notice bar，固定显示「AI空间生成当前仅支持**室内场景**，内容须符合平台规范」

**验收标准**：
- [ ] 仅上传图片（无文字）可正常提交
- [ ] 仅输入文字（无图片）可正常提交
- [ ] 两者均为空时创建按钮 disabled
- [ ] 点创建时有文字则触发检测遮罩
- [ ] 合法检测失败后报错，不提交任务

---

### 自动填写 [P1]

保留现有「✦ 自动填写」按钮，点击后向文字框填入一段示例描述，方便用户参考输入格式。填入后与正常输入一致，AI 扩写按钮同步激活。

## 非功能性需求

| 指标 | 要求 |
|------|------|
| Dify 扩写接口超时 | > 10s 无响应时，停止流式输出，提示「扩写超时，请重试」|
| 扩写内容长度 | 输出不超过 1000 字符（前端截断）|
| 合法检测方式 | 前端关键词匹配（MVP）；后续可升级为后端接口检测 |
| 兼容性 | 与现有创建页一致 |

---

## 数据及埋点需求

| 事件名 | 触发时机 | 关键属性 |
|--------|---------|---------|
| `ai_expand_clicked` | 点击 AI 扩写按钮 | `input_length`, `has_image` |
| `ai_expand_succeeded` | 扩写流式输出完成 | `output_length`, `duration_ms` |
| `ai_expand_failed` | 扩写超时或接口报错 | `error_type` |
| `ai_expand_reverted` | 用户点击撤销 | — |
| `content_check_failed` | 合法检测未通过 | `trigger: "create"\|"expand"` |
| `generation_queued` | 任务提交成功 | `has_text`, `has_image`, `is_expanded` |
